uniapp 官方文档
微信小程序 官方文档

uniapp 的简介

1.uniapp 是可以跨平台的，也就是说一套代码打包可以在很多平台上运行，比如说 微信小程序、支付宝小程序、H5 页面、app 端。（这也是它比较受欢迎的原因）。
2.他的文件类型与微信小程序的相似，比如说pages 页面或者是 .json 配置文件。
3.pages 中的页面第一项是表示启动项，当然后续可以通过页面跳转来看到所有页面。

uniapp 一般组件的使用

1.rpx 与 px 的区别：rpx 是响应式的，是对屏幕大小而变，但 px 是固定的像素。

2.表单组件、媒体组件等可以参考uniapp 参考文档

3.路由跳转组件：
navigator：与微信小程序的 navigatorTo 一样是保留原来页面跳转到新页面。
redirect：与微信小程序的 redirectTo 一样，销毁原来的页面，跳转到新页面。
switchTab：与微信小程序相似，一般用作导航栏跳转，跳转到 tabBar 页面，并关闭其他所有非 tabBar 页面。
reLaunch：与 switchTab 相似，它们的区别 zaizai 与 reLaunch 可以携带参数，switchTab 路径后面不能携带参数，除非有 tabbar。
navigateBack：关闭当前页面，返回上一页
注：如果跳转目的地是 tabbar 页面，那么这些路由跳转会失效。只能是跳转页面内。
注 2：如果想要跳转 tabbar 页面，那么添加属性 open—type：reLaunch

4.form 提交表单：给 form 标签添加 submit 属性绑定事件，给按钮添加 form-type 属性，在事件里得到参数。
5.radio-group 标签 （单选按钮） 6.获取一个表单值有两种方式，一种是在 data 定义一个变量与表单值双向绑定存储表单值。通过事件触发获取表单值，还有就是通过 form>submit 提交表单。
7.picker 标签（滑动块选择）

8.uniapp 组件：uniapp 组件创建和使用非常简单，不像 vue 一样需要注册和引入，uniapp 直接可以创建 components 文件新建组件，然后直接在 pages 页面里面直接用自命名标签的方法去使用组件了，省去了注册和引入。封装的还是比较全面的。

注：props 可以用一个对象，不一定是一个数组，props 接收的值可以通过对象的属性 type、default 来设置传值的类型或者是默认值等等。。。
① 其中，props 传值是单向数据流，父组件可以修改，子组件不允许修改值
② 其中，如果穿的值是数组或者对象，那么必须要用这种方式。如下 ↓

props:{
            title:{
                type:String,
                default:"默认标题"
            }
        }

首页：

<template>
    <view class="box">
        <pubTitle :title="test" subtitle="副标题"></pubTitle>
        <myitem></myitem>

        <h1>uniapp基础</h1>
        <navigator url="/pages/list/list" open-type="reLaunch">跳转到新闻页面</navigator>
        <navigator url="/pages/about/about">跳转到about页面</navigator>
    </view>

</template>

<script>
    export default {
        data(){
            return{
                test:"我是标题"
            };
        }
    }
</script>

<style lang="scss">

    }
    .box h1{
        height: 100px;
        background-color: aqua;
    }
</style>

9.修饰符 native：他的作用是，在组件中子传父的情境中，因为子组件传递了一个事件，在父组件中用子组件就不能用原生的事件了，比如说 click 等。。所以这时候我们就需要修饰符去告诉组件这里需要用到原生事件。–>@click.native=”def”。
